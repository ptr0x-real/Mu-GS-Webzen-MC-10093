- 2003.09.04 (SaemSong) -> (b4nfter)
  User.h 의 OBJECTSTRUCT 구조체는 대량으로 유저와 몬스터 NPC가 사용하기때문에
  메모리를 최대한 아껴야 합니다.  다음과 같은 다리 번호, 방 번호 등 255 바이트
  이내의 데이터는 BYTE 로 하고 크기에 따라 short 형 WORD 형을 사용합니다.
  메모리를 최대한 아낍시다.

  -- 이전
 	int		m_nBloodCastleIndex;	// 블러드 캐슬의 다리번호 (0 ~ 7)
	int		m_nBloodCastleSubIndex;	// 블러드 캐슬의 다리에 몇번째 방에 내가 있는가 (사람일 경우만)
	int		m_iBloodCastleEXP;		// 블러드 캐슬에서 얻은 경험치 (사람일 경우만)
	int		m_iBloodCastleScore;	// 블러드 캐슬에서 얻은 점수 (사람일 경우만)

  -- 이후 (예)
 	BYTE	m_nBloodCastleIndex;	// 블러드 캐슬의 다리번호 (0 ~ 7)
	BYTE	m_nBloodCastleSubIndex;	// 블러드 캐슬의 다리에 몇번째 방에 내가 있는가 (사람일 경우만)
	WORD	m_iBloodCastleEXP;		// 블러드 캐슬에서 얻은 경험치 (사람일 경우만)
	WORD	m_iBloodCastleScore;	// 블러드 캐슬에서 얻은 점수 (사람일 경우만)

------------------------------------------------------------------------------
